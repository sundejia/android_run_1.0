# 补刀搜索查询规范化功能

> 文档创建：2026-02-05
> 状态：已实现
> 最后更新：2026-02-05（简化实现）

## 背景与实现演进

**初始设计**：在步骤3点击搜索结果时，先尝试使用原始名称查找，失败后再使用规范化后的关键词重试。

**当前实现（简化版）**：在步骤3开始时就直接使用规范化后的关键词，避免重复查找操作。这种方式更简洁高效，且逻辑更清晰。

在补刀（FollowUp）系统中，某些客户的名称可能包含后缀标记，例如：

- `B2601300118-(保底正常)`
- `B2601300118-（保底正常）`
- `张三-（VIP客户）`

这些后缀是为了内部管理需要添加的，但在企业微信的搜索功能中：

1. 搜索完整名称（含后缀）可能找不到结果
2. 需要使用简化的主键（如 `B2601300118`）才能准确搜索

## 核心问题

**问题场景**：

```
目标客户名: B2601300118-(保底正常)
搜索输入:   B2601300118-(保底正常)  ❌ 找不到结果
期望搜索:   B2601300118              ✅ 找到结果
```

## 解决方案

### 实现方法

**文件**: `wecom-desktop/backend/servic../03-impl-and-arch/executor.py`

新增 `_normalize_search_query()` 方法：

```python
def _normalize_search_query(self, name: str) -> str:
    """
    规范化"搜索框输入"的关键词。

    需求场景：
    - 补刀目标名可能带后缀：`B2601300118-(保底正常)` 或 `B2601300118-[正常]`
    - 但搜索框需要输入缩减后的主键：`B2601300118`

    规则：
    - 若命中 `-(`, `-（`, `-[`, `-【` 形式，取其前半段
    - 否则返回原字符串（strip 后）
    """
    raw = (name or "").strip()
    if not raw:
        return raw

    # 支持半角/全角括号和方括号
    for sep in ("-(", "-（", "-[", "-【"):
        if sep in raw:
            base = raw.split(sep, 1)[0].strip()
            return base or raw

    # 再兜底：Bxxxxxx-(...) 或 Bxxxxxx-[...] 形式（更严格的正则）
    m = re.match(r"^(B\d+)-\s*[\(（\[【].*$", raw)
    if m:
        return m.group(1)

    return raw
```

### 规范化规则

1. **括号分隔规则**：
   - 检测 `-(半角括号)` 或 `-（全角括号）`
   - 取分隔符前的部分作为规范化后的关键词

2. **正则兜底规则**：
   - 匹配 `B数字-` 开头的格式
   - 提取 `B数字` 部分

3. **默认行为**：
   - 如果没有匹配任何规则，返回原字符串（去除首尾空格）

### 集成点

#### 1. 步骤2：输入搜索关键词

```python
async def _step2_input_search(self, query: str) -> bool:
    """步骤2: 输入搜索关键词"""
    self._log(f"  搜索关键词: [{query}]")

    # 规范化搜索关键词
    normalized = self._normalize_search_query(query)
    if normalized != (query or "").strip():
        self._log(f"  搜索关键词: [{query}] -> 规范化为 [{normalized}]")
    else:
        self._log(f"  搜索关键词: [{normalized}]")

    # 使用规范化后的关键词进行搜索
    await self._input_text(normalized)
    ...
```

#### 2. 步骤3：点击搜索结果

**简化实现（当前版本）**：

```python
async def _step3_click_result(self, target_name: str) -> bool:
    """步骤3: 点击搜索结果"""
    # 直接在步骤开始时就规范化关键词
    normalized = self._normalize_search_query(target_name)
    if normalized != (target_name or "").strip():
        self._log(f"  目标联系人: [{target_name}] -> 规范化为 [{normalized}]")
    else:
        self._log(f"  目标联系人: [{normalized}]")

    # 等待搜索结果加载
    await asyncio.sleep(1.5)
    tree = await self._refresh_ui()

    # 直接使用规范化后的关键词查找（不做兼容重试）
    self._log(f"  在 UI 树中查找 [{normalized}]...")
    results = self._find_elements_by_text(tree, [normalized])

    if results:
        # 找到结果，点击...
        ...

    # 如果没找到精确匹配，尝试点击第一个搜索结果
    self._log(f"  ⚠️ 未找到精确匹配 [{normalized}]", "WARN")
    ...
```

**优势**：

- 逻辑更简单清晰
- 统一使用规范化后的关键词
- 减少重复的查找操作

## 支持的格式

| 原始格式                   | 规范化结果    | 说明       |
| -------------------------- | ------------- | ---------- |
| `B2601300118-(保底正常)`   | `B2601300118` | 半角圆括号 |
| `B2601300118-（保底正常）` | `B2601300118` | 全角圆括号 |
| `B2601300118- (VIP)`       | `B2601300118` | 带空格     |
| `B2601300118-[正常]`       | `B2601300118` | 半角方括号 |
| `B2601300118-【正常】`     | `B2601300118` | 全角方括号 |
| `张三-（VIP客户）`         | `张三`        | 中文姓名   |
| `李四`                     | `李四`        | 无需规范化 |
| ``                         | ``            | 空字符串   |

**注意**：支持 4 种括号类型：`-(`, `-（`, `-[`, `-【`

## 优势

1. **简化逻辑**：
   - 统一在步骤3开始时规范化关键词
   - 避免重复的查找操作
   - 代码更清晰易维护

2. **向后兼容**：
   - 对于没有后缀的名称，规范化后保持不变
   - 不影响现有补刀流程

3. **多格式支持**：
   - 支持半角括号 `-()`
   - 支持全角括号 `-（）`
   - 处理带空格的情况

4. **统一处理**：
   - 步骤2（输入）和步骤3（匹配）都使用规范化后的关键词
   - 确保一致性，减少歧义

5. **清晰日志**：
   - 明确显示规范化前后的关键词
   - 便于调试和问题排查

## 技术细节

### 正则表达式

```python
r"^(B\d+)-\s*[\(（\[【].*$"
```

**解释**：

- `^` - 字符串开始
- `(B\d+)` - 捕获组：B 后跟一个或多个数字
- `-` - 连字符
- `\s*` - 零个或多个空白字符
- `[\(（\[【]` - 半角/全角左圆括号或半角/全角左方括号
- `.*` - 任意字符（用于匹配括号内的内容）
- `$` - 字符串结束

### 分隔符检测

```python
for sep in ("-(", "-（", "-[", "-【"):
    if sep in raw:
        base = raw.split(sep, 1)[0].strip()
        return base or raw
```

**优点**：

- 简单直接，性能好
- 支持 4 种分隔符格式（圆括号和方括号）
- `split(..., 1)` 确保只分割一次
