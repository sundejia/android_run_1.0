# Follow-up 流程分析

## 概述

Follow-up 系统是一个自动化的客户跟进系统，用于在企业微信中自动检测需要跟进的客户并发送跟进消息。系统支持：

- 多设备并行扫描
- 智能消息发送者判断
- AI 自动回复集成
- 定时后台扫描
- 回复检测与标记

## 核心流程

### 1. 触发 Scan Now (手动扫描)

用户点击 "Scan Now" 按钮后的流程：

```
用户点击 "Scan Now"
        │
        ▼
┌───────────────────────────────┐
│ POST /a../03-impl-and-arch/scan       │
│ (followup.py: trigger_scan)   │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ 发现所有已连接设备             │
│ adbutils.adb.device_list()    │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ 创建后台任务                   │
│ asyncio.create_task(          │
│   service.run_scan()          │
│ )                             │
└───────────────┬───────────────┘
                │
                ▼
        返回扫描启动确认
```

### 2. 多设备并行扫描 (run_scan)

```
run_scan()
        │
        ▼
┌───────────────────────────────┐
│ Phase 1: Response Detection   │
│ - 检测有未读消息的用户         │
│ - 为每个用户生成 AI 回复       │
│ - 发送回复并标记已处理         │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ Phase 2: Follow-up Scan       │
│ - 排除 Phase 1 处理过的用户   │
│ - 为每个设备创建扫描任务       │
│ asyncio.gather(               │
│   device_scan_1,              │
│   device_scan_2,              │
│   ...                         │
│ )                             │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ 聚合结果                       │
│ - 合并所有设备的扫描结果       │
│ - 记录日志和统计               │
└───────────────────────────────┘
```

## 消息发送者判断逻辑

### \_is_message_from_kefu() 方法

企业微信聊天界面特点：

- **客服消息**：在屏幕右侧，通常不显示头像
- **客户消息**：在屏幕左侧，左侧显示头像

```
判断流程：
        │
        ▼
┌─────────────────────────────────────────┐
│ 1. 自动检测屏幕宽度                      │
│    _detect_screen_width(all_nodes)      │
│    - 720px / 1080px / 1440px            │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│ 2. 查找消息附近的头像 (im4)              │
│    - 头像在左侧 (< 20% 屏幕宽度) → 客户  │
│    - 头像在右侧 (> 80% 屏幕宽度) → 客服  │
└───────────────────┬─────────────────────┘
                    │
              未找到明确头像
                    │
                    ▼
┌─────────────────────────────────────────┐
│ 3. 查找消息气泡/文本位置                 │
│    - 中心点在右半边 (> 50%) → 客服       │
│    - 中心点在左半边 (< 50%) → 客户       │
└───────────────────┬─────────────────────┘
                    │
              位置不明确
                    │
                    ▼
┌─────────────────────────────────────────┐
│ 4. 没有左侧头像 → 客服                   │
│    (客服消息不显示头像)                  │
└─────────────────────────────────────────┘
```

## 配置选项说明

| 配置项                | 说明                 | 默认值 |
| --------------------- | -------------------- | ------ |
| enabled               | 是否启用跟进系统     | true   |
| scanInterval          | 后台扫描间隔 (秒)    | 60     |
| maxFollowUps          | 每个客户最大跟进次数 | 3      |
| initialDelay          | 首次跟进延迟 (秒)    | 120    |
| subsequentDelay       | 后续跟进延迟 (秒)    | 120    |
| useExponentialBackoff | 使用指数退避策略     | false  |
| backoffMultiplier     | 指数退避倍数         | 2.0    |
| enableOperatingHours  | 限制工作时间         | true   |
| startHour             | 工作开始时间         | 10:00  |
| endHour               | 工作结束时间         | 22:00  |
| useAIReply            | 使用 AI 生成跟进消息 | false  |

### 指数退避策略示例

当 `useExponentialBackoff=true`, `initialDelay=120s`, `backoffMultiplier=2`:

| 跟进次数 | 延迟计算          | 实际延迟     |
| -------- | ----------------- | ------------ |
| 第1次    | initial_delay     | 120s (2分钟) |
| 第2次    | subsequent \* 2^0 | 120s (2分钟) |
| 第3次    | subsequent \* 2^1 | 240s (4分钟) |
| 第4次    | subsequent \* 2^2 | 480s (8分钟) |

## 注意事项

1. **反检测**: 操作之间添加随机延迟，模拟人工操作
2. **并发控制**: 每个设备独立任务，避免操作冲突
3. **错误恢复**: 单用户处理失败不影响其他用户
4. **数据一致性**: 操作结果实时写入数据库
5. **资源释放**: 扫描完成后正确释放连接资源
6. **与全量同步互斥**: 全量同步时需暂停跟进扫描
