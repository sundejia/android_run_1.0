# 新好友聊天记录存储流程分析

## 概述

当检测到新加的好友并点进去聊天后，聊天记录会在 **发送回复之前** 就存入数据库。

## 详细流程

### 1. 新好友检测

新好友的识别是通过消息预览中的关键词来判断的，定义在 `sync_service.py` 和 `unread_detector.py` 中：

```python
NEW_FRIEND_WELCOME_KEYWORDS = (
    # 英文关键词
    "You are now friends",
    "added you as friend",
    "started a conversation",

    # 中文关键词
    "已添加为好友",
    "成为好友",
    "已添加你为好友",
    "开始对话",
    "我通过了你的好友验证",
    # ... 更多关键词
)
```

新好友会被标记为 **高优先级用户**（与有未读消息的用户同等优先级）。

### 2. 完整同步流程

当检测到新好友后，会执行以下流程（定义在 `customer_syncer.py` 的 `sync()` 方法）：

```
┌─────────────────────────────────────────────────────────────────┐
│                     CustomerSyncer.sync()                        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. 获取或创建客户记录                                            │
│     repository.get_or_create_customer(name, kefu_id, channel)   │
│     → 在 customers 表中创建或获取客户记录                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 进入对话                                                      │
│     _enter_conversation(user_name, channel)                      │
│     → 点击用户进入聊天界面（会先捕获头像）                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 提取历史消息                                                  │
│     _extract_messages()                                          │
│     → 从 UI 树中解析所有可见消息                                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. ⭐ 处理并存储每条消息 (发送回复之前)                           │
│     for msg in messages:                                         │
│         _message_processor.process(msg, context)                 │
│         → 调用 repository.add_message_if_not_exists(record)      │
│         → 消息存入 messages 表                                    │
│         → 如果是图片/视频/语音，同时下载媒体文件                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. 交互式回复循环 (如果启用 send_test_messages)                  │
│     _interactive_reply_loop()                                    │
│     → 检查最后一条消息是否是客户发的                               │
│     → 如果是，发送 AI 回复                                        │
│     → 将发送的回复也存入数据库                                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. 退出对话                                                      │
│     _exit_conversation()                                         │
│     → 返回聊天列表界面                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 消息存储流程

消息处理器 (`MessageProcessor`) 会根据消息类型分发到不同的处理器：

```
消息类型          处理器                      存储内容
────────────────────────────────────────────────────────
文本消息    →    TextMessageHandler     →    content, timestamp
图片消息    →    ImageMessageHandler    →    content + 下载图片文件
表情包      →    StickerMessageHandler  →    content (标记为 sticker)
语音消息    →    VoiceMessageHandler    →    content + 下载语音文件
视频消息    →    VideoMessageHandler    →    content + 下载视频文件
```

每种处理器最终都会调用：

```python
added, msg_record = self._repository.add_message_if_not_exists(record)
```

### 5. 消息去重机制

数据库使用 `message_hash` 进行去重：

```python
def add_message_if_not_exists(self, message: MessageRecord) -> Tuple[bool, MessageRecord]:
    msg_hash = message.message_hash or message.compute_hash()
    message.message_hash = msg_hash

    existing = self.get_message_by_hash(msg_hash)
    if existing:
        return False, existing  # 已存在，跳过

    # 不存在，创建新记录
    created = self.create_message(message)
    return True, created
```

Hash 计算规则（定义在 `models.py`）：

- 基于 `customer_id + content + timestamp_raw + is_from_kefu + message_type`
- 这样即使重复同步同一用户，也不会产生重复消息

## 总结

| 问题                         | 答案                                                              |
| ---------------------------- | ----------------------------------------------------------------- |
| 聊天记录什么时候存入数据库？ | **发送回复之前**                                                  |
| 具体时机                     | 进入聊天界面 → 提取历史消息 → **立即存入数据库** → 然后才发送回复 |
| 发送的回复会存入吗？         | 会的，发送成功后会调用 `_store_sent_message()` 存入               |
| 重复同步会产生重复消息吗？   | 不会，有基于 hash 的去重机制                                      |

## 数据库表结构

主要涉及的表：

- `customers` - 客户信息
- `messages` - 消息记录
- `images` - 图片文件路径
- `videos` - 视频文件路径
- `voice_messages` - 语音文件路径（在 extra_info 中）
