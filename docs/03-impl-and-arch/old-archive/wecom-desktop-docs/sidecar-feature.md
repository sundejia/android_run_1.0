# Sidecar 功能说明

## 概述

Sidecar（侧边栏/辅助窗口）是一个**人工审核消息发送**的功能模块，它在自动化同步过程中提供一个中间层，让用户可以在消息自动发送之前进行审核、编辑或取消。

## 启用方式

在 Settings（设置）页面中：

1. 勾选 **"Send via Sidecar"** 启用 Sidecar 模式
2. 设置 **"Countdown Seconds"**（倒计时秒数，默认10秒）
3. 设置 **"Sidecar Poll Interval"**（轮询间隔，默认10秒，0=禁用自动轮询）

## 核心目的

### 1. 消息发送前审核

- 在自动同步过程中，系统会将待发送的消息**先放入队列**
- 用户可以在 Sidecar 界面中**查看、编辑或取消**这些消息
- 提供**倒计时机制**，在指定时间内用户可以干预

### 2. 防止误发送

- 自动化程序可能会生成不适当的回复
- Sidecar 提供了一个**安全检查点**
- 用户可以在消息发出前进行最后确认

### 3. AI 回复集成

- 当启用 AI 回复时，AI 生成的消息会先进入 Sidecar 队列
- 用户可以查看 AI 生成的内容并进行修改
- 支持查看消息来源（AI、Mock、Fallback）

## 工作流程

```
同步扫描客户消息
        ↓
   需要回复?
        ↓ 是
[send_via_sidecar = true]
        ↓
  消息加入队列
        ↓
 Sidecar 界面显示
        ↓
   倒计时开始
        ↓
用户可: 编辑/取消/立即发送
        ↓
  倒计时结束
        ↓
   自动发送消息
```

## 主要功能

### 消息队列管理

| 功能     | 说明                               |
| -------- | ---------------------------------- |
| 添加消息 | 同步过程中自动添加待发送消息到队列 |
| 查看队列 | 显示所有待处理的消息               |
| 暂停队列 | 暂停自动发送，手动控制             |
| 恢复队列 | 继续自动发送流程                   |
| 取消队列 | 取消所有待发送消息                 |

### 倒计时机制

- **可配置倒计时时长**（0-30秒）
- **自动检测设备输入**：如果检测到设备上有手动输入，自动暂停倒计时
- **手动干预**：用户可以随时暂停、编辑或取消

### 多设备支持

- 支持同时打开**最多3个设备**的 Sidecar 面板
- 每个设备独立管理自己的消息队列
- 支持拖拽添加设备面板

## 界面功能

### 1. 设备状态显示

- 当前对话信息（联系人名称、渠道）
- 客服信息
- 同步状态

### 2. 消息编辑区

- 显示待发送消息内容
- 支持实时编辑
- 显示消息来源标识（AI/Mock/Fallback）

### 3. 控制按钮

- **Start Countdown**：开始倒计时
- **Send Now**：立即发送
- **Pause**：暂停队列
- **Resume**：恢复队列
- **Cancel**：取消当前消息

### 4. 历史记录

- 显示与当前联系人的对话历史
- 支持折叠/展开
- 可调整高度

### 5. 日志流

- 实时显示设备操作日志
- 支持折叠/展开
- 可调整高度

## 使用场景

### 场景1：批量同步时的安全检查

当需要同步大量客户消息时，启用 Sidecar 可以：

- 逐条审核自动生成的回复
- 发现问题时立即取消
- 确保回复内容的准确性

### 场景2：AI 回复的人工审核

当使用 AI 自动回复时：

- AI 生成的回复先进入 Sidecar
- 用户审核 AI 回复是否合适
- 必要时进行编辑修正

### 场景3：敏感客户处理

对于重要或敏感客户：

- 启用较长的倒计时时间
- 仔细审核每条回复
- 确保沟通质量

## 配置选项

| 设置项                | 说明                  | 默认值  |
| --------------------- | --------------------- | ------- |
| `sendViaSidecar`      | 是否启用 Sidecar 模式 | `false` |
| `countdownSeconds`    | 倒计时时长（秒）      | `10`    |
| `sidecarPollInterval` | 状态轮询间隔（秒）    | `10`    |

## API 端点

| 端点                                 | 方法 | 说明              |
| ------------------------------------ | ---- | ----------------- |
| `/sidecar/{serial}/state`            | GET  | 获取 Sidecar 状态 |
| `/sidecar/{serial}/send`             | POST | 发送消息          |
| `/sidecar/{serial}/queue`            | GET  | 获取消息队列      |
| `/sidecar/{serial}/queue/add`        | POST | 添加消息到队列    |
| `/sidecar/{serial}/queue/ready/{id}` | POST | 标记消息为就绪    |
| `/sidecar/{serial}/queue/send/{id}`  | POST | 发送队列中的消息  |
| `/sidecar/{serial}/queue/pause`      | POST | 暂停队列          |
| `/sidecar/{serial}/queue/resume`     | POST | 恢复队列          |
| `/sidecar/{serial}/queue/cancel`     | POST | 取消队列          |

## 与直接发送模式的对比

| 特性       | 直接发送模式     | Sidecar 模式     |
| ---------- | ---------------- | ---------------- |
| 发送速度   | 快               | 较慢（需要审核） |
| 人工干预   | 无               | 支持             |
| 适用场景   | 已测试验证的回复 | 需要审核的回复   |
| 错误风险   | 较高             | 较低             |
| 操作复杂度 | 简单             | 需要关注界面     |

## 总结

Sidecar 功能的核心价值在于**在自动化和人工控制之间取得平衡**：

- 保留自动化的效率
- 添加人工审核的安全性
- 支持灵活的干预机制
- 适用于需要谨慎处理的业务场景
