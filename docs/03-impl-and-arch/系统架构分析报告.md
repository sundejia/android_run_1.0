### 6.2 中优先级问题

#### **问题 4: 错误处理不统一**

**当前状况**:

```python
# 有些地方返回 None
def get_customer(id: int) -> Optional[Customer]:
    try:
        return db.query(...)
    except Exception:
        return None

# 有些地方抛出自定义异常
def get_device(serial: str) -> Device:
    device = db.query(...)
    if not device:
        raise DeviceNotFoundError(serial)
    return device

# 有些地方抛出 HTTPException
@router.get("/devices/{serial}")
async def get_device(serial: str):
    device = manager.get_device(serial)
    if not device:
        raise HTTPException(status_code=404, detail="Device not found")
```

**建议**: 统一错误处理策略

```python
# 1. 定义应用异常层次
class WecomException(Exception):
    """应用基础异常"""

class DeviceNotFoundError(WecomException):
    """设备未找到"""

class CustomerNotFoundError(WecomException):
    """客户未找到"""

# 2. 全局异常处理器
@app.exception_handler(WecomException)
async def wecom_exception_handler(request, exc):
    return JSONResponse(
        status_code=400,
        content={"error": exc.message, "type": type(exc).__name__}
    )

# 3. 服务层抛出业务异常
def get_device(serial: str) -> Device:
    device = db.query(Device).filter_by(serial=serial).first()
    if not device:
        raise DeviceNotFoundError(f"Device {serial} not found")
    return device
```

**影响**: 📈 **中等** - 影响错误追踪和调试
**优先级**: 🟡 **中**

```python
# schema.py 返回 str

# 但 core/config.py 返回 Path
@property
def db_path(self) -> Path:
    return Path(self._db_path)

# 使用时需要转换
db_path = get_db_path(None)  # 返回 Path
if db_path.exists():         # ✅ Path 方法
    conn = sqlite3.connect(str(db_path))  # 需要转 str
```

**建议**: 统一使用 `Path` 对象

```python
from pathlib import Path

DEFAULT_DB_PATH: Path = Path("wecom_conversations.db")

def get_db_path(custom_path: Optional[str] = None) -> Path:
    if custom_path:
        return Path(custom_path)
    return DEFAULT_DB_PATH

# sqlite3.connect 支持 Path 对象 (Python 3.6+)
conn = sqlite3.connect(get_db_path())
```

**影响**: 📈 **低** - 小幅提升代码质量
**优先级**: 🟢 **低**

### 6.3 低优先级改进

#### **改进 1: 添加依赖注入容器**

**当前**: 手动依赖管理

```python
# 手动创建和传递依赖
class WeComService:
    def __init__(self, config: Config):
        self.adb = ADBService(config.adb)
        self.parser = UIParserService(config.ui_parser)
        self.db = DatabaseService(config.database)
```

**建议**: 使用 DI 容器

```python
from dependency_injector import containers, providers

class Container(containers.DeclarativeContainer):
    config = providers.Configuration()

    adb_service = providers.Factory(
        ADBService,
        config=config.adb
    )

    ui_parser = providers.Factory(
        UIParserService,
        config=config.ui_parser
    )

    wecom_service = providers.Factory(
        WeComService,
        adb=adb_service,
        parser=ui_parser
    )

# 使用
container = Container()
wecom = container.wecom_service()
```

**优点**:

- 自动依赖解析
- 易于测试 (可替换实现)
- 配置集中管理

**优先级**: 🟢 **低** - 可选的架构改进
